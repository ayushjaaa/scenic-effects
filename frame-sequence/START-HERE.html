üéØ How It Works (Like Rotoris Website)

The Rotoris website uses individual images, not video:

1. Individual Images

100+ separate image files (AVIF, JPG, PNG, WebP)

2. Preload All

Load all images in background before starting

3. Scroll Control

Scrolling changes which image displays

4. Smooth Animation

JavaScript interpolates between frames

Not a video! Each "frame" is a separate image file. When you scroll, JavaScript changes the image source, creating the illusion of video playback.

‚öôÔ∏è Setup Guide (For Real Images)

Step 1: Prepare Your Images

Option A: Extract from Video

./extract-frames.sh -i your-video.mp4 -f 30 -q 85

Option B: Generate Test Images

mkdir frames && cd frames # Then use ImageMagick to generate test images (see QUICKSTART.md)

Option C: Download from URLs

./download-frames.sh -u 'https://example.com/frame_{NUM}.avif' -s 1 -e 120

Step 2: Verify Your Files

Make sure you have this structure:

frame-sequence/
‚îî‚îÄ‚îÄ frames/
    ‚îú‚îÄ‚îÄ frame_00001.jpg
    ‚îú‚îÄ‚îÄ frame_00002.jpg
    ‚îú‚îÄ‚îÄ frame_00003.jpg
    ‚îî‚îÄ‚îÄ ... (100+ images)

Step 3: Configure Settings

Open avif-sequence.js and update:

const CONFIG = {
  imageBaseURL: './frames/frame_',
  imageFormat: 'jpg',        // Your file format
  totalFrames: 120,          // Number of images
  paddingZeros: 5,           // For 00001 format
};

Step 4: Start Web Server

Important: You must use a web server!

python3 -m http.server 8000

Then visit: http://localhost:8000/frame-sequence/avif-sequence.html

‚ö†Ô∏è Don't open files directly! Using file:// won't work due to browser security. Always use a web server.
